# Repository Guidelines

## Project Structure & Module Organization

Diamond X is a mobile-first static SPA/PWA backed by Supabase. The app shell is `index.html`, with PWA metadata in `manifest.json` and offline caching in `service-worker.js`. Frontend modules live in `js/`: `app.js` handles bootstrapping and hash routing, shared auth/API helpers sit in `auth.js`, `supabase.js`, `ui.js`, `calendar.js`, `qrcode.js`, and role-specific screens are under `js/pages/{admin,student,responsible}/`. CSS is split into `css/reset.css`, `css/variables.css`, `css/components.css`, and `css/pages.css`. Static media and web fonts are in `assets/`; database changes are in numbered `migrations/`; Supabase Edge Functions are in `supabase/functions/`; browser tests and reports are in `testsprite_tests/`.

## Build, Test, and Development Commands

This repository has no npm build pipeline; `package.json` only marks ES module mode.

- `python3 -m http.server 8080`: serve the app locally for manual QA at `http://localhost:8080`.
- `python3 -m http.server 3000`: serve the app on the port expected by existing TestSprite Playwright tests.
- `python3 testsprite_tests/TC001_Register_and_land_in_the_role_based_area.py`: run one generated browser test; replace the filename for other cases.
- `deno check supabase/functions/asaas-checkout/index.ts`: type-check a Supabase Edge Function; repeat for changed functions.
- `supabase functions deploy <function-name>`: deploy Edge Functions after local validation.

## Coding Style & Naming Conventions

Use plain JavaScript ES modules with named exports where practical. Match the existing style: 4-space indentation, semicolons, single quotes for strings, `camelCase` for variables/functions, and descriptive module names such as `trainingReservations.js`. Keep UI text in Portuguese to match the product. CSS should reuse tokens from `css/variables.css` and place shared components in `components.css`; page-specific rules belong in `pages.css`.

## Testing Guidelines

Tests are Python Playwright scripts generated by TestSprite in `testsprite_tests/`, named `TC###_Description.py`. Keep tests deterministic, role-specific, and aligned with hash routes such as `/#dashboard` or `/#register`. Before running them, serve the static app on port `3000` and ensure Supabase test configuration supports the scenario.

## Commit & Pull Request Guidelines

Recent commits use short Portuguese summaries, for example `ajuste do scroll` and `melhorias realizadas, solicitadas pelo Ricardo Diamond`. Follow that concise imperative style and keep each commit focused. Pull requests should describe the user-facing change, list affected roles, mention Supabase migration/function changes, link any relevant issue or spec, and include screenshots or short screen recordings for UI changes.

## Security & Configuration Tips

Do not commit real Supabase keys, service role tokens, or Asaas secrets. Review RLS-sensitive changes in `migrations/` together with frontend role checks in `js/auth.js` and route rendering in `js/app.js`.
